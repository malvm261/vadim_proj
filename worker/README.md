# Worker

Отдельный uv-проект воркера для сервера `Pi Distributed Master`.

Воркер подключается к мастеру, регистрируется, забирает чанки через
`/workers/health`, выполняет переданную сервером функцию `compute(start, end)` и
отправляет прогресс и итоговый частичный результат обратно на сервер.

## Запуск

Из папки `worker`:

```bash
uv run main.py --server-url http://127.0.0.1:8000
```

Или из корня сервера:

```bash
uv run --project worker worker/main.py --server-url http://127.0.0.1:8000
```

Полезные параметры:

```bash
uv run main.py \
  --server-url http://127.0.0.1:8000 \
  --worker-id worker-1 \
  --progress-step 10000
```

`--progress-step` задаёт, на какие поддиапазоны воркер делит полученный чанк,
чтобы регулярно отправлять `/workers/progress`.

## Несколько процессов на одной машине

По умолчанию `main.py` запускает столько процессов-воркеров, сколько ядер CPU
на машине (`--processes`, он же `-p`). Каждый процесс регистрируется на
мастере отдельно (`<worker-id>-0`, `<worker-id>-1`, ...) и забирает свои чанки
независимо — поэтому одна машина считает в N раз быстрее, используя все ядра:

```bash
uv run main.py --server-url http://127.0.0.1:8000 --processes 8
```

Чтобы вернуться к старому однопроцессному поведению:

```bash
uv run main.py --server-url http://127.0.0.1:8000 --processes 1
```

## Устойчивость к сетевым сбоям

Запросы к мастеру (`register`, `health`, `progress`, `result`) повторяются с
нарастающей паузой при временных сетевых ошибках или ответах 5xx, поэтому
короткий обрыв связи не приводит к потере уже посчитанного чанка. Сбой
отправки промежуточного прогресса не прерывает расчёт — он лишь логируется,
а финальный результат чанка отправляется с повторами до победного.
