# Distributed Compute

Распределённый расчёт: один **сервер** (мастер) раздаёт куски задачи произвольному
числу **воркеров**, агрегирует частичные результаты и даёт управлять всем
процессом через веб-интерфейс в браузере. Чем больше подключено и включено
воркеров — тем быстрее считается задача.

Сервер и воркер — независимые uv-проекты (`server/` и `worker/`), их можно
запускать на разных машинах.

## Требования

- Python ≥ 3.14
- [uv](https://docs.astral.sh/uv/)

## 1. Запуск сервера

Из папки `server`:

```bash
cd server
uv run main.py
```

По умолчанию сервер поднимается на `http://0.0.0.0:8000`. Веб-интерфейс
доступен в браузере по адресу `http://127.0.0.1:8000` — там можно ввести код
функции `compute(start, end)`, параметры задачи (число итераций, число
чанков), запустить/остановить/продолжить расчёт и включать/выключать каждый
подключённый воркер.

Хост и порт настраиваются через `.env` в папке `server` (или переменные
окружения), см. `app/config.py`:

```env
HOST=0.0.0.0
PORT=8000
```

## 2. Запуск воркера

Из папки `worker`, указав адрес сервера:

```bash
cd worker
uv run main.py --server-url http://<адрес-сервера>:8000
```

Воркер сам регистрируется на сервере, забирает чанки задачи, считает их и
отправляет результаты обратно. Можно запускать сколько угодно воркеров — на
этой же машине или на других — каждый из них появится в веб-интерфейсе
сервера и начнёт получать работу, как только задача запущена.

По умолчанию воркер запускает по процессу на каждое ядро CPU машины (флаг
`--processes` / `-p`), чтобы использовать все ядра — это даёт кратное
ускорение расчёта на одной машине. Подробности по флагам, многопроцессному
режиму и устойчивости к сетевым сбоям — в [`worker/README.md`](worker/README.md).

Пример запуска нескольких воркеров с разных машин на один сервер:

```bash
# машина A
uv run main.py --server-url http://192.168.1.10:8000 --worker-id node-a

# машина B
uv run main.py --server-url http://192.168.1.10:8000 --worker-id node-b
```

## Структура проекта

```
server/   — FastAPI-сервер (мастер): API, веб-интерфейс, координатор задачи
worker/   — клиент-воркер: забирает чанки, считает, шлёт результаты
```
